# F-flat-minor Codetta - compiled-byte scores

This folder holds the shipped Coda Etudes used by Codetta. For Codetta, the competitive score is the size of the optimized compiled program: **optimized `.ffb` bytes**.

Reference source metrics are still tracked alongside that score:

- **Bytes:** length of the source file in UTF-8 bytes.
- **Chars:** number of Unicode code points in the source.

For Codetta, though, the score that matters is the optimized compiled size, because the live web UI, README `bytes:` metadata, and current-best comparisons all use compiled output rather than source length.

## Scorecard

[`SCORECARD.md`](SCORECARD.md) is regenerated by [`score_codetta.py`](score_codetta.py). It walks each `ff/codetta/<slug>/` folder, finds `solution.ff` or `solution.ffp`, and records:

- source **bytes** and **chars**,
- **optimized `.ffb` bytes**: the Codetta score,
- README frontmatter **bytes** for cross-checking.

The optimized compiled-byte score is the length of compiler stdout from [`shell/ff-compile.sh`](../shell/ff-compile.sh) with optimization enabled. That count includes the `FbAbbCb` header plus the base64 bytecode payload, matching the `.ffb` layout used by the web app.

From the repository root:

```bash
python3 ff/codetta/score_codetta.py
```
